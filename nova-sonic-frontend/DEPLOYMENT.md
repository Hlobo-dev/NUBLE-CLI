## Step 1: Create Route 53 Hosted Zone

Run these commands to set up DNS:

```bash
# Create hosted zone for cloudlobo.com
aws route53 create-hosted-zone \
  --name cloudlobo.com \
  --caller-reference "cloudlobo-$(date +%s)" \
  --hosted-zone-config Comment="Nova Sonic Debug Production"
```


## Step 3: Request SSL Certificate (ACM)

```bash
# Request certificate for cloudlobo.com and *.cloudlobo.com
aws acm request-certificate \
  --domain-name cloudlobo.com \
  --subject-alternative-names "*.cloudlobo.com" \
  --validation-method DNS \
  --region us-east-1
```

Then validate via DNS:
```bash
# Get the certificate ARN from above command, then:
aws acm describe-certificate \
  --certificate-arn arn:aws:acm:us-east-1:456309723884:certificate/XXXXX \
  --region us-east-1
```

Add the CNAME record to Route 53 for validation (ACM will show you the values).

## Troubleshooting

### DNS not resolving
```bash
# Check nameservers
dig cloudlobo.com NS

# Check A record
dig cloudlobo.com A
```

### Certificate not validating
- Make sure CNAME validation record is in Route 53
- Wait up to 30 minutes for validation

### WebSocket disconnecting
- Increase ALB idle timeout to 4000s
- Enable sticky sessions on target group
